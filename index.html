AI Manifesto

A reference manifesto for AI usage across my projects. It defines how AI is used, for what purpose, and what to expect regarding the project's evolution.

Vibe-Coding

All or nearly all of the code and associated architectural decisions in this project were generated by AI, with little to no human review.

Expectations

  • The developer may have little to no understanding of the code.
  • The project may not follow software engineering best practices.
  • External contributions are not expected. If you want to evolve this project, it is recommended to fork it.
  • There is no guarantee of long-term support and/or maintenance.

AI-Assisted Development

All or nearly all of the code in this project is generated via AI, and the majority is reviewed by the developer. Architectural decisions are made by the developer.

Expectations

  • The developer has a strong understanding of the code.
  • The developer has a strong understanding of the software architecture.
  • The project strives to follow software engineering best practices.
  • External contributions may be accepted. The project may be equipped with a harness to facilitate AI-assisted contributions.

AI Manifesto

Un manifeste de référence pour l'utilisation de l'IA dans mes projets. Il définit comment l'IA y est utilisée, dans quel but ainsi que les attentes d'évolutions associées au projet.

Vibe-Coding

La totalité ou quasi-totalité du code ainsi que les décisions architecturales associées à ce projet ont été générées par IA, avec pas ou peu de revue.

Attentes

  • Le développeur peut avoir une compréhension inexistante ou incomplète du code.
  • Il se peut que ce projet ne suive pas les meilleures pratiques en matière de développement logiciel.
  • Les contributions externes ne sont pas prévues. Dans le cas où vous voudriez faire évoluer ce projet, il est recommandé de le fourcher.
  • Aucune garantie de support et/ou de maintenance sur le long terme.

Développement assisté par IA

La totalité ou quasi-totalité du code associé à ce projet est généré via IA et une majeure partie est revue par le développeur. Les décisions architecturales sont définies par le développeur.

Attentes

  • Le développeur a une forte compréhension du code.
  • Le développeur a une forte compréhension de l'architecture du logiciel.
  • Le projet essaie de suivre les meilleures pratiques en matière de développement logiciel.
  • Les contributions externes peuvent être acceptées, le projet peut être équipé d'un harnais permettant les contributions assistées par IA.
